# Requirement Capture Examples

`example-engineering-requirements.json` is a fictional, manually reviewed capture of thesis formatting requirements. It maps approved requirements to `ThesisFormatSpec` and `TemplatePackage` paths before a template is considered for publication.

`example-engineering-requirements-invalid.json` intentionally uses an absolute source path and an approved unmapped requirement for validator and CLI negative tests.

Evidence fields are short references only. Do not paste long copyrighted passages into requirement capture files.
