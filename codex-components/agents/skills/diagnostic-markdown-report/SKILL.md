# Diagnostic Markdown Reports

Markdown reports are for humans; JSON is the machine contract.

Rules:

- Keep Markdown concise.
- Prioritize status, blocking issues, top actions, checklist, merge decision, and first fix hints.
- Do not include sensitive content or long source excerpts.
- Markdown should be suitable for PR comments.
- Keep JSON reports stable for automation.
