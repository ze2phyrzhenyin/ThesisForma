# Onboarding Workspace

- Keep real institution source material in private `onboarding-workspaces/<slug>/` folders.
- `examples/onboarding` must stay fictional.
- Do not implement AI parsing; scaffold commands create manual-entry files only.
- All workspace paths must be relative and must not contain `..`.
- Update schemas, tests, docs, and CLI help when changing onboarding fields.
